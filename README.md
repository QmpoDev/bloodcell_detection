# EventDP — Blood Cell Image Classification

Event-Driven Programming lab project: image classification on blood cell types using a prepared subset of the [Blood Cell Images](https://github.com/Shenggan/BCCD_Dataset) dataset (dataset2-master).

## Project structure

```
EventDP/
├── data_raw/              # Original dataset (dataset2-master: augmented TRAIN + TEST)
├── data_split/            # Train / validation / test split (4000 images, generated by script)
│   ├── TRAIN/             # 2800 images (70%)
│   ├── VAL/               # 600 images (15%)
│   └── TEST/              # 600 images (15%)
├── prepare_data.py        # Builds data_split from data_raw (Lab 2 data preparation)
├── check_data_split.py    # Validates data_split counts and structure
├── training.py            # CNN training (TensorFlow/Keras); saves to saved_model/
├── classify.py            # Classify a single image with the trained model
├── augment.py             # Optional data augmentation (camera/xray); requires Pillow
├── README.md
└── Paul Emmanuelle Quimpo - Laboratory 2 - Data Preparation.pdf
```

## Dataset

- **Source:** Blood Cell Images (dataset2-master) — augmented images (rotated, scaled, flipped).
- **Classes:** Eosinophil, Lymphocyte, Monocyte, Neutrophil (4 classes).
- **Subset (default):** 4000 images (1000 per class), 70% / 15% / 15%.
- **Full dataset:** `prepare_data.py --full` uses all images (~12.5k): 70% of TRAIN → train, 30% → val, 100% of TEST → test.
- Train/val from original TRAIN folder; test from TEST folder only (no leakage).

### data_split layout (4000 subset)

| Set       | Images | Per class |
|----------|--------|-----------|
| TRAIN    | 2800   | 700       |
| VAL      | 600    | 150       |
| TEST     | 600    | 150       |
| **Total**| **4000**| **1000**  |

For full dataset, counts depend on `data_raw` (~6.9k train, ~3k val, ~2.5k test).

## Setup

1. Place the original dataset so that this path exists:
   - `data_raw/dataset2-master/dataset2-master/images/`
   - with subfolders `TRAIN/` and `TEST/`, each containing class folders: `EOSINOPHIL`, `LYMPHOCYTE`, `MONOCYTE`, `NEUTROPHIL`.

2. (Optional) Create a virtual environment:
   ```bash
   python -m venv venv
   venv\Scripts\activate   # Windows
   ```

3. For **training**: install TensorFlow and Pillow (e.g. `pip install tensorflow pillow`). Keras uses Pillow to load images; **augment.py** also needs Pillow.

## Cloning and setting up on another PC

On a new machine, clone the repo and prepare the environment and data:

```bash
# 1. Clone the repository
git clone https://github.com/QmpoDev/bloodcell_detection.git
cd bloodcell_detection

# 2. (Optional) Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate   # Windows
# source venv/bin/activate   # Linux / macOS

# 3. Install dependencies
pip install tensorflow pillow

# 4. Add the dataset: place the dataset2-master folder so this path exists:
#    data_raw/dataset2-master/dataset2-master/images/
#    with subfolders TRAIN/ and TEST/, each containing EOSINOPHIL, LYMPHOCYTE, MONOCYTE, NEUTROPHIL.
#    (Get the dataset from the Blood Cell Images / BCCD source; it is not in the repo.)

# 5. Build the train/val/test split (choose one)
python prepare_data.py              # 4000-image subset
# python prepare_data.py --full     # full dataset (~12.5k)

# 6. (Optional) Verify the split
python check_data_split.py          # for 4000 subset
# python check_data_split.py --full # for full dataset

# 7. Train the model
python training.py
```

After training, the model is saved as `saved_model/blood_cell_model.keras`. Use `python classify.py path/to/image.jpg` to classify an image.

## Usage

### Data preparation

Build the train/validation/test split from the raw data.

**4000-image subset (default):**
```bash
python prepare_data.py
```
- 1000 per class, 70% train / 15% val / 15% test (2800 / 600 / 600).

**Full dataset (~12.5k):**
```bash
python prepare_data.py --full
```
- Uses all images: 70% of TRAIN → train, 30% of TRAIN → val, 100% of TEST → test.
- Uses a fixed random seed (42); run again to regenerate (overwrites `data_split/`).

### Verify data split

**4000 subset:** check structure and counts (2800 / 600 / 600):
```bash
python check_data_split.py
```

**Full dataset:** report counts only:
```bash
python check_data_split.py --full
```

### Training

Run training (auto-detects 4000 or full-dataset sizes; model saved to `saved_model/blood_cell_model.keras`):

```bash
python training.py
```

Uses the professor-provided CNN (4-class softmax for blood cell types). Validation is done on VAL; final accuracy is reported on TEST.

### Classify an image

After training, classify a single image with the saved model:

```bash
python classify.py path/to/image.jpg
```

Or run `python classify.py` and enter the image path when prompted. Prints the predicted class (Eosinophil, Lymphocyte, Monocyte, Neutrophil) and confidence.

### Data augmentation (optional, not used in this pipeline)

Our **training images are already augmented** at the source (dataset2-master: rotated, scaled, flipped), so we do not run any augmentation script or on-the-fly augmentation in `training.py`.

**augment.py** is an optional, professor-provided tool for a *different* case: when you have another dataset with **class imbalance** (some folders have fewer images than others). It creates augmented copies (flips, rotations, color) until every class has as many images as the largest class. Use it only if you need to balance such a dataset:

```bash
pip install pillow
python augment.py
```

When prompted, enter the dataset path and image type (`camera` or `xray`). Use `camera` for blood cell / microscope images.
